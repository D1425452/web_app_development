# 產品需求文件 (PRD)

## 專案名稱
食譜收藏系統

## 專案描述
這是一個專屬於個人的食譜收藏與管理系統，讓使用者能夠輕鬆新增、編輯、尋找食譜，並留下烹飪心得，打造屬於自己的數位食譜庫。

---

## 1. 專案概述

- **背景與動機**：平時在各地看到想要嘗試的食譜，或是家傳的秘方，往往散落在各處不易整理。為了在下廚時能夠快速找到需要的食譜，並隨時記錄下每一次烹飪的心得，需要開發一個專屬個人的集中化食譜管理系統。
- **目標用戶**：我自己（個人使用者）
- **核心價值主張**：提供一個輕量、簡單直覺的操作介面，不但能用圖文完整記錄食譜，還能透過標籤及關鍵字搜尋迅速找到目標菜色，並將試錯的筆記記錄下來，優化每次的烹飪體驗。

## 2. 功能需求

以下為系統核心的主要功能：

1. **食譜圖文紀錄**
   - 作為使用者，我希望可以新增帶有圖片的食譜內容（包含食材、步驟），以便完整保留菜色看起來的樣子與做法。

2. **標籤分頁**
   - 作為使用者，我希望可以為食譜加上特定的標籤（例如：中式、甜點、備餐），並透過標籤分頁瀏覽，以便我能依據當下想煮的類別快速尋找所需食譜。

3. **關鍵字搜尋**
   - 作為使用者，我希望可以輸入關鍵字搜尋食譜標題或內容，以便在累積大量食譜後，仍能快速定位某一道特定的菜餚。

4. **我的最愛**
   - 作為使用者，我希望可以將常做的食譜加入「我的最愛」，以便能隨時打開常用清單，不用每次都重新篩選或搜尋。

5. **烹飪筆記**
   - 作為使用者，我希望能在每個食譜中編寫或更新「烹飪筆記」，以便記錄這次烹調的小錯誤或是創意改良，讓下次下廚時表現更好。

## 3. 非功能需求

- **技術限制**：
  - 後端與路由：Python Flask
  - 前端渲染：Jinja2 樣板引擎 (搭配 HTML/CSS/JS)
  - 資料庫：SQLite (確保輕量、免安裝資料庫伺服器即可運行，適合個人專案)
- **效能與安全性考量**：
  - 圖片處理：上傳圖片需注意大小與格式限制，以避免塞爆儲存空間並保持頁面載入速度。
  - 安全防護：使用 ORM（如 SQLAlchemy）或參數化查詢來防止基本的 SQL Injection；在 Jinja2 預設轉義的保護下避免 XSS 攻擊。

## 4. MVP 範圍 (Minimum Viable Product)

| 優先級 | 功能項目 | 說明 |
| --- | --- | --- |
| **Must Have** | 基本 CRUD | 建立新增、查看（列表與詳細）、編輯、刪除食譜的功能 |
| **Must Have** | 關鍵字搜尋 | 能以文字搜尋食譜標題 |
| **Should Have** | 食譜圖文紀錄 | 支援圖片上傳與文字編輯 |
| **Should Have** | 烹飪筆記 | 每個食譜具備獨立的文字區塊記錄個人筆記 |
| **Nice to Have**| 標籤分頁 | 提供標籤管理並具備過濾篩選列表的功能 |
| **Nice to Have**| 我的最愛 | 提供星號按鈕以收藏食譜，並獨立出一個篩選頁面 |
